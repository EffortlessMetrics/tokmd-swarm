# tokmd 1.14 Release Ledger

## Purpose

tokmd 1.14 is the PR evidence-packet workflow release. It promotes the
`sensors/tokmd/` evidence packet from a manual multi-step recipe into a
first-class local and CI workflow via `tokmd render`, `tokmd packet generate`,
and the `mode: packet` GitHub Action. It also sharpens syntax-receipt and
complexity evidence and makes several CLI error and validation paths more
actionable.

This ledger is the maintainer release record. It is intentionally narrow: the
1.14 cycle does not change any receipt schema version, add a reachability or
safety proof, or perform release mutation from `tokmd-swarm`. Tagging, GitHub
release creation, crates.io publication, and GHCR publication happened from the
publication repository `EffortlessMetrics/tokmd`.

### Post-release install smoke

A clean `cargo install tokmd --version 1.14.0` from crates.io (into an isolated
`--root`) built and installed `tokmd v1.14.0` on Windows MSVC. The installed
binary reported `tokmd 1.14.0`, and `tokmd render --help` and
`tokmd packet generate --help` both resolved, confirming the new packet-workflow
CLI surface is present in the **default** install feature set (the
`v1.13.1`-style default-feature regression did not recur). This smoke proves the
crate installs and exposes the documented commands; it does not exercise a full
packet run or prove runtime correctness.

## Lane Summary

| Lane | Outcome | User impact |
| --- | --- | --- |
| Packet rendering CLI | Added `tokmd render` to render packet presets (including unsafe-review bundles) from analysis/context/syntax inputs, ingest sibling tool bundles, and validate against the `tokmd-packets` JSON Schema at the CLI boundary. | Consumers can build validated packet presets from one command. |
| Packet orchestrator CLI | Added `tokmd packet generate`, a one-command evidence-packet orchestrator that produces the `sensors/tokmd/` packet (analysis, context, optional syntax, manifest). | The evidence packet is boring to generate locally. |
| `mode: packet` Action | Added `mode: packet` to the `EffortlessMetrics/tokmd` GitHub Action with prebuilt-binary runtime default, base-ref inference, packet artifact upload, a job summary, and packet status/manifest/dir/count outputs plus `fail-on` status mapping. | CI can produce and gate on evidence packets without bespoke steps. |
| Syntax receipt evidence | Added `panic_seam_summary` to syntax receipts and deprioritized test-assertion noise in syntax evidence ranking. | Packet consumers get higher-value advisory review signals. |
| Complexity correctness | Aligned `max_cyclomatic` with per-function complexity units. | The reported maximum reflects a single function, not an aggregate. |
| CLI actionability | Made `tokmd diff` `resolve_targets` errors actionable, added numeric-flag validation for `tokmd context` / `tokmd handoff`, and detect UTF-16 JSON inputs (bare PowerShell `>` redirect) in packet/evidence-packet ingestion. | Common CLI mistakes now produce clear, fixable errors. |
| Security | Upgraded `pyo3` (and its PyO3 crate family) `0.28.3 → 0.29.0` in `tokmd-python`, resolving `RUSTSEC-2026-0176` flagged by `cargo deny`. | The Python binding dependency surface clears the advisory. |
| Docs | Added usage example blocks to `tokmd module`, `run`, and `init` help and regenerated `docs/reference-cli.md`. | `--help` shows common invocations consistently. |
| CI economics | Retired the duplicate routed Rust Small frontdoor workflow and its lane catalogue entries; routing is now the advisory `Route CI runner` job feeding the single required `Tokmd Rust Result` gate. | No required check changed; the duplicate lane is gone. |
