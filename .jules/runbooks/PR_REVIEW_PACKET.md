# PR Review Packet

Make review boring. Make truth cheap.

## 💡 Summary
1–4 sentences. What changed.

## 🎯 Why
What problem or friction existed, and why this change is worth reviewer attention.

## 🔎 Evidence
Minimal proof:
- file path(s)
- observed behavior or issue
- command/test/bench/doctest/fuzz/mutation receipt demonstrating it

## 🧭 Options considered
### Option A (recommended)
- What it is
- Why it fits this repo and this shard
- Trade-offs: Structure / Velocity / Governance

### Option B
- What it is
- When to choose it instead
- Trade-offs

## ✅ Decision
State the decision and why.

## 🧱 Changes made (SRP)
Bullets with file paths.

## 🧪 Verification receipts
Copy from the run packet.

## 🧭 Telemetry
- Change shape
- Blast radius (API / IO / docs / schema / concurrency / compatibility)
- Risk class + why
- Rollback
- Gates run

## 🗂️ .jules artifacts
What per-run packet was written.
What friction items or persona notes were created.

## 🔜 Follow-ups
If anything remains, create friction items and link them.
